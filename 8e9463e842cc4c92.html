The Core Problem

Current AI Memory Captures
Outcomes, Not Process

The latest memory systems from Anthropic and OpenAI embed text summaries of what happened. They don't capture the reasoning process, the decision points, or the consequences that led to each outcome.

RAG-Based Memory

How everyone else does it
Stores text summaries: "Patient prefers morning appointments" or "Treatment plan was updated"
Embeddings on text summaries, not on the reasoning process. One vector per memory entry.
Records that a treatment plan was created. Not why that specific combination was chosen over alternatives.
Can store "avoid Drug X with Drug Y" as a rule. Can't learn the diagnostic pattern that led to discovering that interaction.

Episodic Memory

The LLABS approach
Captures the full clinical reasoning trajectory: lab analysis, drug interaction checks, treatment selection, patient response
Hierarchical structure: patient engagements, clinical sessions, reasoning iterations, individual EMR queries and actions
Four named vectors per experience. "Show me every time the agent reasoned about lipid panel abnormalities" is a single search.
When a protocol changes or a contraindication is discovered, that specific learning is updated. Everything else stays.
Technical Architecture

How Episodic Memory Works

Every time the agent works on something, the system captures the full decision-making process. Not a summary. The actual trajectory.

01

Loop Decomposed

Each agentic reasoning loop stored as an independent record with its own embeddings

02

Actions Recorded

Every tool call, its parameters, and its result linked to the reasoning that triggered it

03

Results Captured

Environmental responses, user feedback, error states. The full consequence chain.

04

Trajectory Encoded

Each iteration embedded into multi-dimensional vector representations and stored in a searchable index

The Technical Innovation

Four-Dimensional
Experience Encoding

Most systems store one embedding per experience. We store four. Each one captures a different dimension of how the agent thought and what happened.

Dimension 01

Combined Trajectory

Reasoning + outcome concatenated. "Find all past sessions related to prior art analysis." General similarity across 15 patent analysis sessions.

Dimension 02

Reasoning Patterns

Thinking process only. Same 15 sessions now cluster by analytical approach: claim construction, prior art search, and obviousness analysis. Outcome doesn't matter here.

Dimension 03

Outcome Space

Results only. Same sessions arranged by what actually happened on a continuous spectrum. Missed references on one end, valid arguments on the other, partial outcomes in between. Not binary.

Dimension 04

Contextualized

Post-engagement re-embedding. Each iteration re-encoded with awareness of every other step in the same patent engagement. Lines show sibling relationships.

The Breakthrough

Recursive Trajectory Learning

When the system gets feedback, it extracts learnings and stores them as persistent knowledge. That knowledge is then automatically surfaced alongside every related experience in all future sessions. The agent can also go back and audit its own history.

vs. Reinforcement Learning

Beyond Scalar Rewards

Both RL and episodic trajectory learning aim to improve agent performance over time. The approach is fundamentally different. RLHF gives you a scalar score or preference ranking. It tells the model something was wrong, but not what, why, or how to fix it.

Reinforcement Learning

RLHF / DPO / PPO
Reward signal: scalar score or preference pair. No context for what to change.
Needs thousands of examples to shift model behavior distribution
Modifies base model weights. Expensive compute, risk of capability regression.
Training errors bake into the weights permanently. Corrections require retraining.
No reasoning trace. Can't explain its own decision process.
Unlearning a specific behavior requires retraining on modified data

Episodic Trajectory Learning

The LLABS approach
Full contextual feedback. The system knows what to change and why.
Learns from a single interaction. One piece of feedback creates knowledge that surfaces alongside every related future session.
No weight modification. Runs on top of any cloud model. Model-agnostic.
Fix one specific learning without affecting anything else. Deprecate, update, or supersede.
Full reasoning trace stored at every decision point. Auditable.
Agent searches its own trajectory history, compares past reasoning, refines its knowledge base.
