Model-Agnostic Architecture
Our cognitive architecture brings domain-specific capabilities to any underlying model. Switch to the latest models as they release while retaining all extraction accuracy and domain training. The intelligence layer is separate from the base model, so upgrades are seamless.
All Endpoints
Method Endpoint Description
POST /v1/extract Synchronous extraction. Returns JSON immediately.
POST /v1/extract/async Async extraction. Returns job_id for polling.
GET /v1/extract/jobs/{job_id} Poll async job status and retrieve results.
POST /v1/extract/batch Batch extraction (up to 10 documents).
GET /v1/schemas List available pre-built schemas.
GET /v1/schemas/{id} Get full schema definition.
GET /health Health check for load balancers.
Examples
1 Using a Pre-built Schema (DOT Bridge Plan)

Use a pre-built schema ID instead of defining your own. The dot_bridge_plan schema extracts all standard fields from DOT bridge construction documents.

  • Response includes field_details with per-field confidence scores and citations
  • instructions provides natural language guidance to focus on specific sections
  • Each citation includes page number and exact source text
Request
{
  "document": "JVBERi0xLjQK...",
  "schema": "dot_bridge_plan",
  "instructions": "Focus on permit tables",
  "options": {
    "include_citations": true,
    "confidence_threshold": 0.85
  }
}
Response
{
  "status": "completed",
  "data": {
    "bridge_type": "Multi-span (5 spans)",
    "jurisdiction": "Noxubee County, MS",
    "total_length_ft": 500.00,
    "permits": { ... }
  },
  "field_details": {
    "bridge_type": {
      "confidence": 0.98,
      "citation": { "page": 1,
        "text": "2@99' and 3@100'" }
    },
    "jurisdiction": {
      "confidence": 0.99,
      "citation": { "page": 1,
        "text": "Noxubee County, Mississippi" }
    },
    "total_length_ft": {
      "confidence": 0.97,
      "citation": { "page": 1,
        "text": "Length: 500.00 FT" }
    }
  },
  "overall_confidence": 0.97
}
2 Custom Schema with Nested Objects

Define your own schema to extract exactly what you need. Supports nested objects and arrays.

  • Use description to guide extraction for ambiguous fields
  • Arrays extract all matching items (e.g., all line items in a table)
Request
{
  "document": "JVBERi0xLjQK...",
  "schema": {
    "vendor": {
      "type": "object",
      "properties": {
        "name": { "type": "string" },
        "address": { "type": "string" }
      }
    },
    "line_items": {
      "type": "array",
      "items": {
        "description": { "type": "string" },
        "quantity": { "type": "number" },
        "unit_price": { "type": "number" }
      }
    },
    "total": { "type": "number" }
  }
}
Response
{
  "status": "completed",
  "data": {
    "vendor": {
      "name": "ABC Construction",
      "address": "123 Main St, Jackson MS"
    },
    "line_items": [...],
    "total": 79500
  },
  "field_details": {
    "vendor.name": {
      "confidence": 0.99,
      "citation": { "page": 1,
        "text": "ABC Construction LLC" }
    },
    "total": {
      "confidence": 0.96,
      "citation": { "page": 2,
        "text": "Grand Total: $79,500.00" }
    }
  },
  "overall_confidence": 0.94
}
3 Async Batch Processing (Construction Bids)

Process multiple construction bid documents in a single request. Each result includes full extracted data with per-field confidence and citations.

  • Up to 10 documents per batch request
  • Each document gets its own id for tracking in results
  • Poll /v1/extract/jobs/{job_id} until status is completed
  • Each result includes the same field_details structure as single extractions
POST /v1/extract/batch
{
  "documents": [

  "schema": "construction_bid",
  "options": { "include_citations": true }
}

4 Using Document URL (S3/HTTP)

Instead of base64-encoding the document, provide a URL. Works with S3 pre-signed URLs or any HTTP endpoint.

  • Use document_url instead of document
  • For S3, generate a pre-signed URL with read access
  • Reduces payload size for large documents
Request
{
  "document_url": "https://your-bucket.s3.amazonaws.com/docs/bridge_plan.pdf?X-Amz-...",
  "schema": "dot_bridge_plan",
  "options": {
    "include_citations": true,
    "include_bounding_boxes": true
  }
}
Response
{
  "status": "completed",
  "data": {
    "total_cost": 2450000,
    "contractor": "ABC Construction"
  },
  "field_details": {
    "total_cost": {
      "confidence": 0.98,
      "citation": {
        "page": 3,
        "text": "Total: $2,450,000",
        "bounding_box": {
          "x": 120, "y": 450,
          "width": 200, "height": 20
        }
      }
    }
  },
  "overall_confidence": 0.97
}
